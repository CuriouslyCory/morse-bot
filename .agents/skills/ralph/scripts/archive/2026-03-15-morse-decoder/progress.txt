## Codebase Patterns
- This is a pnpm monorepo using Turborepo. Apps in apps/, shared packages in packages/, tooling in tooling/.
- sherif@latest enforces alphabetical ordering in package.json dependencies/devDependencies. Always sort keys alphabetically.
- pnpm postinstall runs sherif lint:ws automatically; fix ordering issues before committing.
- Package scope is @morse-bot/ (was @acme/ in the t3-turbo template).
- TypeScript imports must NOT use .js extensions — project uses bundler module resolution.
- The Expo app uses expo-router for file-based routing (similar to Next.js App Router).
- The Expo app uses NativeWind (Tailwind CSS for React Native) — use className prop on RN components.
- tRPC client is configured at apps/expo/src/utils/api.tsx — use `trpc` for query/mutation options.
- Auth client is configured at apps/expo/src/utils/auth.ts — use `authClient.useSession()`, `authClient.signIn.social()`, `authClient.signOut()`.
- @morse-bot/morse-decoder is pure JS and works in React Native without changes.
- @morse-bot/ui (Radix-based) does NOT work in React Native — build RN equivalents with NativeWind classes.
- requestAnimationFrame and cancelAnimationFrame are available in React Native.
- All tRPC mutations/queries that access user-owned data must include userId in where clause.
- @morse-bot/db re-exports drizzle-orm helpers (eq, and, desc, etc.).
- The path alias ~/ maps to apps/expo/src/ in the Expo app (configured in tsconfig.json).
- Use Pressable (not TouchableOpacity) for interactive elements — it's the modern RN approach.
- For portable functions (dbToColor, buildCharTimings, elementsToSymbols, formatDuration), copy directly from the Next.js app — they are pure math/string functions with no web dependencies.
