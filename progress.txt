## Codebase Patterns
- Package scope: @moris-bot (renamed from @acme)
- All hooks use `useRef` + `onSamplesRef.current = onSamples` pattern to keep callbacks fresh — this triggers `react-hooks/refs` ESLint errors in custom hooks but is the established pattern in this codebase
- Pre-existing lint errors (19 in original, reduced to 13): react-hooks/refs in use-audio-input.ts, use-morse-decoder.ts; do not add new lint errors
- `useRef<boolean>(false)` required (not `useRef(false)`) to avoid TypeScript narrowing `current` to literal `false`, which would cause `@typescript-eslint/no-unnecessary-condition` errors on async cancellation checks
- Async cancellation checks across await boundaries (`if (cancelledRef.current)`) need `// eslint-disable-next-line @typescript-eslint/no-unnecessary-condition` since TypeScript can't reason about ref mutations across awaits
- AudioContext sample rate is 8000 Hz (matches live mic input) for consistent decoder behavior
- Timestamps for file decoding: derived from sample position `(offset / audioBuffer.sampleRate) * 1000` — not wall-clock time
- Existing hooks (useAudioInput, useMorseDecoder) handle their own internal ref syncing; decoder-panel.tsx should NOT add an additional onSamplesRef wrapper
